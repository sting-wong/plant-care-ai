# 产品原则 (Product Principles)

「植物管家 Plant Care AI」的设计与决策原则。每一轮迭代、每一个改动都以此为准绳。与原则冲突的需求，先质疑再实现。

## 一、核心是「帮人把植物养活」，不是功能清单

- 优先保证核心流程跑通：拍照/选图 → AI 诊断 → 保存植物 → 提醒养护。
- 任何断裂的核心流程，优先级高于一切新功能。
- 砍掉用不上的功能比加功能更有价值。

## 二、给普通养植物的人用，不是给植物学家

- 用户面文案用中文、口语化，避免学名堆砌（学名作为补充而非主信息）。
- 默认值要聪明：AI 推荐浇水/施肥周期，用户能改但不必改。
- 减少必填项，能推断的就别让用户填。

## 三、数据一致、可信

- 同一份数据（如健康状态、品种、置信度）在 scan → diagnosis → save → 首页之间必须一致，不能各算各的。
- AI 不确定时如实表达不确定，不编造精确数字。
- gardenScore 等汇总指标要反映真实状态（健康差就该扣分）。

## 四、移动端体验优先

- 触控目标足够大，关键操作一步可达。
- 上传照片同时提供「拍照」和「从相册选择」，不强制单一入口。
- 加载、识别、保存都有明确反馈（loading / 成功 / 失败）。

## 五、隐私与安全不妥协

- API key 永不进源码、永不进日志，只存 `.env.local`。
- 用户照片与数据本地优先（localStorage / Zustand persist），不偷偷外传。

## 六、克制与一致的视觉语言

- 统一 Glasmorphism 玻璃拟态风格（glass-bg / glass-card / glass-panel）。
- 绿色主色 `#2D7D46`，文案灰 `#6B7B6B`，深色字 `#1A2E1A`。
- 复用现有组件与 CSS 类，不随意引入新风格或新依赖。
