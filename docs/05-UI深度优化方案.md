# UI 深度优化方案 - 视觉丰富度提升

## 问题诊断

当前 UI 的核心问题：
1. **图片缺失** — 植物百科用 emoji 代替图片，社区帖子没有图片，整体视觉单薄
2. **图标太素** — 全部用 lucide-react 线条图标，缺乏品牌感和层次
3. **页面层次平** — 卡片之间缺少视觉节奏，没有大小对比、色彩对比
4. **装饰元素少** — 没有背景纹理、渐变、装饰性图形，页面太"干净"
5. **空间利用不够** — 很多地方可以用视觉元素填充，增加丰富感

---

## 一、植物图片系统

### 方案：使用高质量 SVG 植物插画

由于没有后端存储真实照片，用精致的 SVG 插画代替。每种植物一个专属插画，比 emoji 高级很多。

// src/components/post-image.tsx

// 根据帖子的 plantTag 生成对应的装饰性配图
export function PostImage({ plantTag, seed }: { plantTag?: string; seed: number }) {
  // 生成一个带植物插画 + 柔和背景色的配图
  const bgColors = [
    "from-green-100 to-emerald-50",
    "from-amber-50 to-yellow-50",
    "from-blue-50 to-cyan-50",
    "from-pink-50 to-rose-50",
  ];
  const bg = bgColors[seed % bgColors.length];

  return (
    <div className={`w-full aspect-[4/3] rounded-xl bg-gradient-to-br ${bg} flex items-center justify-center`}>
      {/* 根据 plantTag 显示对应的植物插画 */}
      <PlantIllustrationByName name={plantTag} className="w-24 h-24 opacity-80" />
    </div>
  );
}
```

## 六、首页视觉层次优化

- 拍照入口太大，挤占了信息空间

### 改造方案

```
首页视觉层次（从上到下）：

   - 背景用绿色渐变 + 装饰性植物剪影

   ┌──────────────────────────┐

   └──────────────────────────┘

4. 今日待办（保持现有，但加入进度条）
   "今日 2/5 已完成" [━━━━━━━━░░]

5. 季节卡片（加入配图）
