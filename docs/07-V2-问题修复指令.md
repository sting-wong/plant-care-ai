## 问题 2：AI 回复带思考过程 + 上下文丢失

### 根因分析

当前用的模型是 `glm-5v-turbo`（不是我们文档里推荐的 `glm-4v-plus`）。这个模型是带推理能力的（类似 DeepSeek-R1），它会输出 `reasoning_content`（思考过程）和 `content`（最终回答）。

代码第 71-73 行的处理逻辑有问题：

```typescript
const message = data.choices?.[0]?.message;
const content = message?.content || message?.reasoning_content || "";
```

这里的逻辑是：如果 `content` 为空就用 `reasoning_content`。但问题是 `glm-5v-turbo` 可能把最终回答放在 `content` 里，思考过程放在 `reasoning_content` 里，也可能两个都有内容但 `content` 包含了思考过程标记。

### 修复方案

**方案 A（推荐）：换回不带推理的模型**

把模型从 `glm-5v-turbo` 换成 `glm-4v-plus`。`glm-4v-plus` 不会输出思考过程，直接给最终回答。

```typescript
// zhipu-service.ts 第 55 行
model: "glm-4v-plus",  // 改回 glm-4v-plus，不用推理模型
```

**方案 B：如果要继续用 glm-5v-turbo，需要过滤思考内容**

```typescript
async function callZhipu(messages: ZhipuMessage[]): Promise<string> {
  const apiKey = process.env.ZHIPU_API_KEY;

  if (!apiKey) {
    throw new Error("ZHIPU_API_KEY not configured");
  }

  const response = await fetch("https://open.bigmodel.cn/api/paas/v4/chat/completions", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "Authorization": `Bearer ${apiKey}`,
    },
    body: JSON.stringify({
      model: "glm-4v-plus",  // 换回 glm-4v-plus
      messages,
      max_tokens: 500,
      temperature: 0.7,
    }),
  });

  if (!response.ok) {
    const errorText = await response.text();
    throw new Error(`智谱 API 错误 ${response.status}: ${errorText.slice(0, 200)}`);
  }

  const data = await response.json();

  const message = data.choices?.[0]?.message;
  let content = message?.content || "";
  
  // 过滤掉思考过程标记（如果有的话）
  // 有些模型会用 <think>...</think> 或类似标记包裹思考过程
  content = content.replace(/<think>[\s\S]*?<\/think>/g, "").trim();
  content = content.replace(/\[思考过程\][\s\S]*?\[\/思考过程\]/g, "").trim();
  
  // 如果过滤后为空，尝试用 reasoning 之外的内容
  if (!content && message?.reasoning_content) {
    // reasoning_content 是思考过程，不应该展示给用户
    // 这种情况说明模型没有生成最终回答，给一个兜底
    content = "让我再想想...你能补充一下更多信息吗？";
  }

  return content || "抱歉，我暂时无法回复。";
}
```

### 上下文丢失问题

根因：`conversations` 是一个内存 Map，存在 Next.js 的服务端。但 Next.js 的 API Route 可能在不同的 serverless 实例上运行，或者 HMR 热更新时 Map 会被清空。

**修复方案：把对话历史通过前端传递，不依赖服务端内存**

修改 `src/app/api/diagnosis/chat/route.ts`，让前端把完整对话历史传过来：

```typescript
// src/app/api/diagnosis/chat/route.ts
import { NextResponse } from "next/server";
import { aiService } from "@/lib/ai";

export async function POST(request: Request) {
  try {
    const body = await request.json();
    const { conversationId, message, history } = body;

    // 如果前端传了 history，用它来恢复上下文
    const result = await aiService.chat({ 
      conversationId, 
      message,
      history,  // 新增：前端传入的对话历史
    });

    return NextResponse.json({ reply: result });
  } catch (error: unknown) {
    const errMsg = error instanceof Error ? error.message : String(error);
    console.error("Chat error:", errMsg);
    return NextResponse.json(
      { reply: "网络好像不太稳定，再试一次？" },
      { status: 200 }  // 不返回 500，让前端能正常显示兜底消息
    );
  }
}
```

修改 `ChatRequest` 类型：

```typescript
// src/lib/ai/types.ts
export interface ChatRequest {
  conversationId: string;
  message: string;
  history?: { role: string; content: string }[];  // 新增
}
```

修改 `zhipu-service.ts` 的 chat 方法：

```typescript
async chat(req: ChatRequest): Promise<string> {
  // 优先使用前端传来的历史，其次用内存中的
  let history: ZhipuMessage[] = [];
  
  if (req.history && req.history.length > 0) {
    // 从前端传来的历史恢复
    history = [
      { role: "system", content: SYSTEM_PROMPT },
      ...req.history.map(h => ({ role: h.role as "user" | "assistant", content: h.content })),
    ];
  } else {
    // 尝试从内存获取
    history = conversations.get(req.conversationId) || [
      { role: "system" as const, content: SYSTEM_PROMPT },
    ];
  }

  history.push({ role: "user", content: req.message });

  const aiText = await callZhipu(history);

  history.push({ role: "assistant", content: aiText });
  conversations.set(req.conversationId, history);

  return aiText;
}
```

修改前端 `diagnosis/[id]/page.tsx` 的 `handleSend`，把对话历史一起传过去：

```typescript
const handleSend = async (message: string) => {
  addMessage(sessionId, {
    id: nanoid(8),
    role: "user",
    content: message,
    timestamp: Date.now(),
  });

  setIsTyping(true);
  try {
    // 构建历史上下文传给后端
    const history = session.messages.map(m => ({
      role: m.role,
      content: m.content,
    }));

    const res = await fetch("/api/diagnosis/chat", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        conversationId: session?.diagnosis?.conversationId || sessionId,
        message,
        history,  // 传入完整对话历史
      }),
    });
    const data = await res.json();

    addMessage(sessionId, {
      id: nanoid(8),
      role: "assistant",
      content: data.reply,
      timestamp: Date.now(),
    });
  } catch {
    addMessage(sessionId, {
      id: nanoid(8),
      role: "assistant",
      content: "网络好像不太稳定，再试一次？",
      timestamp: Date.now(),
    });
  } finally {
    setIsTyping(false);
  }
};
```
